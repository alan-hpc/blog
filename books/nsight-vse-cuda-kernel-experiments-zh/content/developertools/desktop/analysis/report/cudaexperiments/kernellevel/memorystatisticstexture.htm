内存统计 — 纹理内存

NVIDIA® Nsight™ Development Platform, Visual Studio Edition 4.7 用户指南
发送反馈

概述

纹理内存是只读的设备内存，可以使用 Texture Functions 中描述的设备函数进行访问。使用其中一个函数读取纹理称为 texture fetch（纹理取数）。纹理内存的流量经由纹理缓存（独立于 L1 数据缓存）和 L2 缓存路由。纹理内存系统在设计上对二维数组中具有空间局部性的访问效率较高。GPU 的纹理硬件支持还提供了超出典型内存系统的特性，例如越界读取时的可定制行为，以及在数组元素坐标之间读取时的过滤插值。纹理内存的设计目标是以恒定延迟进行流式取数；纹理缓存命中可以减少设备内存带宽占用，但不会降低取数延迟。

图表

纹理（Texture）

每当一个 warp 执行一次纹理函数来从纹理内存读取时，就计为一次取数（fetch）。Texture 图表显示取数次数和取数速率。一次 texture fetch 是一次内存请求，会从纹理缓存中产生事务。来自纹理缓存的事务以 32 字节为单位。

注意，本图表不包括全局只读数据访问（使用 const __restrict__ 或 __ldg() intrinsic），后者同样使用纹理缓存。对于这些访问，请参见 内存统计 — 全局内存。

分析

  • 如果纹理缓存命中率较低……
    • ……说明访问模式与纹理缓存针对二维空间局部性所做的优化不匹配。尝试修改访问模式，使空间上相邻的读取更紧凑地聚集在一起。或者，对于一维线性访问模式，全局内存可能比纹理内存更高效。
  • 如果纹理内存带宽成为性能瓶颈……
    • ……考虑使用更简单的纹理过滤模式，或更低分辨率的纹理。
