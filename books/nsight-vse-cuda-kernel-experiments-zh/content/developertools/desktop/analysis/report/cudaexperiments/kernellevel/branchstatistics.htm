分支统计

NVIDIA® Nsight™ Development Platform, Visual Studio Edition 4.7 用户指南
发送反馈

概述

控制流可能对 kernel 的执行效率产生严重影响，尤其是当大量控制流判定都是分歧的，迫使一个 warp 中的线程在 kernel 代码中走上完全不同的控制路径时。分支统计实验有助于回答以下问题：控制流指令被执行了多少次、其中有多少是一致的、有多少是分歧的，以及控制流对整体 kernel 执行性能的影响有多大。

背景

如果一条控制流指令迫使 warp 中的线程走上不同的执行路径，则该指令被视为分歧的。一旦发生这种情况，由于一个 warp 中的所有线程共享一个程序计数器，不同的执行路径必须被串行化执行；这会增加该 warp 所执行的指令总数。当所有不同的执行路径都完成后，线程才会重新汇合到同一条执行路径。如果条件表达式对一个 warp 中的所有线程评估出一致的判定结果，则只执行被选中的那一条代码路径——因此开销小得多。所执行的一致控制流判定数与全部条件判定执行数之比定义为分支效率（Branch Efficiency）。

分歧控制流造成的实际性能影响，与需要评估多少条不同代码路径以及被串行化的代码段有多昂贵的组合成正比。一种刻画方法是：跟踪所有已执行指令中，warp 内实际参与执行（即未被谓词执行屏蔽掉）的线程数。这通常被称为控制流效率（Control Flow Efficiency）。按定义，它与做出多少次控制流判定无关，而是给出了由于控制流而导致的可用计算资源利用率的上限。

效率

效率图显示了用于评估控制流影响的两个主要指标。重点关注控制流效率，以便在考虑分支的情况下估计预期的最大峰值性能。

用 CUDA C 编写的条件表达式可以通过多种方式被翻译成最终的汇编代码。请注意，并非所有可能的条件操作变体都能被这里展示的指标涵盖。因此可以这样说：控制流效率低一定意味着存在性能问题；但反之不一定成立——即使一个 kernel 的控制流效率接近最优，它仍然可能受到控制流的限制。

指标

分支效率 表示所有已执行分支指令中一致控制流判定所占的比例。按 SM 分别显示（柱状图），并对所有 SM 取平均（Branch 折线）。数值越高越好，意味着 warp 更频繁地走在一致的代码路径上。低于 100% 的值是 kernel 执行性能受到负面影响的必要但不充分指标，因为该指标并不知道条件语句所包围的代码区域有多大。例如，十次执行的分支中有一次分歧判定，若它只包围寥寥几行代码，则可以忽略不计；但如果它迫使 warp 执行多条包含数千条指令的不同代码路径，影响就可能非常大。

控制流效率 定义为：在每条已执行指令上，未被谓词执行屏蔽的活跃线程数与每个 warp 最大线程数之比。每条指令中每个 warp 内活跃的线程数越少，该指标就越低；因此它衡量的是对可用处理单元的利用效率。控制流效率较低可能由以下原因导致：启动 warp 时活跃线程数不足 32；warp 中部分线程比其他线程更早终止；或者只让一部分线程参与指令的执行。

每 warp 分支数

按条件语句的判定结果分组，显示每个 SM 上每个 warp 平均执行的分支指令数。可用于研究 kernel grid 中各 warp 所执行的控制流指令总量。将每 warp 执行的分支指令数与 指令统计 实验中 IPW 指标所报告的每 warp 已执行指令总数进行对比，可以了解 kernel 代码中执行分支指令的频率。

所报告的 taken / not taken 分支数指的是在设备上实际执行的条件语句。如果编译器在优化阶段选择了对条件取反，这并不一定与 CUDA C 或 PTX 代码中表达的定义一致。可查看 源码视图 页面以研究高级代码与汇编代码之间的映射关系。

指标

Not Taken / Taken 每 warp 平均执行的、控制流判定一致的分支指令数；即一个 warp 中所有活跃线程要么全部走分支，要么全部不走。

Diverged 每 warp 平均执行的、条件在 warp 内各线程间产生不同结果的分支指令数。所有至少有一个线程参与的代码路径都会被顺序执行。该数值越低越好；不过，要理解控制流对设备利用率的影响，请同时查看 控制流效率。

分析

CUDA C Best Practices Guide 中重点强调了关于 控制流 的若干通用编码准则：

  • 分支与分歧：避免同一 warp 内出现不同的执行路径。
  • 分支谓词执行：让编译器更容易使用分支谓词执行来替代循环或控制语句。
  • 循环计数器：有符号 vs. 无符号：循环计数器使用有符号整数而非无符号整数。
  • 在循环中同步分歧线程：避免在分歧的代码内部使用 __syncthreads()。

根据本实验的结果，检查以下哪些情况适用：

  • 如果 控制流效率 较低，并且……
    • ……分支效率 较高，请检查 kernel 启动的 执行配置。如果每个 block 的线程数不是 warp 大小的整数倍，部分活跃的 warp 只使用了可用处理单元的一部分——从而降低整体控制流效率。
    • ……分支效率 也较低，则 kernel 执行因控制流分歧而效率较低。请在可能的情况下重新安排负载，使一个 warp 中的线程都走同一条路径。使用 分歧分支 实验识别哪些分支指令导致的分歧最严重、影响的代码区域最大，优先优化这些分支。
